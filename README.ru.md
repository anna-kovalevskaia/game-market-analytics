# Game Market Analytics System (via Open-Source Data)

Аналитическая система для исследования игрового рынка на основе открытых данных. Построена как сквозная data platform: от сырых API до интерактивных дашбордов с ежедневным обновлением.

## Цель

Исследовать исторические данные игровой индустрии и проанализировать, как метрики популярности, жанровые характеристики и медийный интерес соотносятся с косвенными коммерческими показателями игр - в разрезе студий разного масштаба.

## Исследовательские вопросы

- **Инвестиционные сигналы** - Каковы косвенные маркеры коммерческой целесообразности для разных жанров на текущем рынке? Каков порог видимости для игр разного масштаба?
- **Рыночная концентрация** - Как распределён пользовательский трафик внутри популярных жанров? Жанр действительно доступен для новых игроков или его метрики завышены за счёт 2-3 многолетних хитов?
- **Жанровые тренды** - Как менялся интерес аудитории к ключевым жанрам и тегам за последние 3 года?
- **Аномалии рынка (Black Swans)** - Выявление кейсов, добившихся высокой популярности вопреки общим трендам или при минимальных стартовых ресурсах.
- **Экономика внимания (Attention Decay Rate)** - Анализ связи между часами просмотров на Twitch и динамикой онлайна в Steam, оценка скорости затухания медийного хайпа.
- **Влияние масштаба студии** - Плотность релизов и удержание аудитории в разрезе категорий: крупные издатели, средний сегмент, инди.

## Источники данных

### Фаза 1 - MVP (текущая)

| Источник | Что даёт | Почему |
|---|---|---|
| Steam Web API / Store API | Метаданные игр, ценовые тиры, CCU | Официальный, стабильный, бесплатный |
| SteamSpy | Оценка owners, прокси выручки | Индустриальный стандарт для Steam-аналитики |
| IGDB API | Структурированная классификация жанров, рейтинги критиков | Steam-теги неструктурированные; IGDB даёт чистую таксономию |
| Twitch API | Часы просмотров, вовлечённость | Измеряет медийный вес игровых тайтлов |

### Фаза 2 - Планируется

| Источник | Что даёт |
|---|---|
| SteamDB (scraping) | Историческая динамика цен и скидочных кампаний |
| Epic Games Store API | История бесплатных раздач и их влияние на метрики в Steam |
| RAWG.io API | Кроссплатформенные данные (консольный рынок) |
| Itch.io API / RSS | Инди-экосистема и детекция зарождающихся микро-трендов |
| Reddit & YouTube API | Тональность и активность сообществ вокруг игр |
| Google Trends API | Динамика глобального поискового интереса |

## Технологический стек

| Инструмент | Роль | Почему, а не альтернативы |
|---|---|---|
| Apache Airflow | Оркестрация | Python-native DAG с управлением зависимостями, ретраями, UI и алертингом. Выбран вместо Prefect (официальная интеграция с dbt заброшена в 2024) и Dagster (менее распространён в вакансиях). Богатая экосистема провайдеров. |
| ClickHouse | Хранилище данных | Колоночная OLAP СУБД для субсекундных агрегаций на append-only данных временных рядов. PostgreSQL заточен под OLTP и медленен на аналитических запросах. Облачные решения (AWS, Databricks) - избыточны без публичного ассистента, добавляются в фазе 3 когда публичный доступ потребует инфраструктуры. |
| dbt | Слой трансформаций | Data-as-Code: Staging -> Core -> Marts с автогенерацией lineage-графа, документации, Jinja-макросов и тестов качества данных. SQL модели читаемы и поддерживаемы в отличие от raw ETL скриптов. |
| Google Sheets | Облачный буфер | Лёгкий бесплатный слой для публикации готовых под дашборд витрин из локального DWH в BI без серверного бэкенда. В облако уходят только агрегированные, готовые под дашборд витрины. |
| Tableau Public | Визуализация | Подключён к Google Sheets для автоматического ежедневного обновления публичного дашборда. |
| Azure Static Web Apps | Хостинг фронтенда | Публичный хостинг для embedding дашборда через iframe. |

## Архитектура

```
[Steam API]  [SteamSpy]  [IGDB]  [Twitch API]
       |           |        |          |
       +-----------+--------+----------+
                       |
                  [Airflow DAGs]
                  (Python ETL, ежедневное расписание,
                   ретраи, алертинг)
                       |
              [ClickHouse - локальный DWH]
              (raw -> staging -> core -> marts)
                       |
                    [dbt]
              (трансформации, тесты,
               документация, lineage)
                       |
          [Google Sheets - облачный буфер]
          (только готовые под дашборд витрины)
                       |
       [Tableau Public - живой дашборд]
                       |
       [Azure Static Web Apps - iframe]
```

**Принцип архитектуры: Serverless & Local-First BI**

Вся тяжёлая вычислительная работа (ETL, dbt-материализации, колоночное сжатие) выполняется локально в ClickHouse. В облако отправляются только компактные, готовые под дашборд витрины. Стоимость инфраструктуры - $0, дашборд доступен публично.

## Этапы разработки

**Фаза 1 - Local-First MVP** <- текущая
- Steam API + SteamSpy + IGDB + Twitch -> ClickHouse
- Airflow DAGs с ежедневным расписанием, ретраями, алертингом
- dbt модели: staging -> core -> marts
- Google Sheets как облачный буфер
- Tableau Public с автоматическим ежедневным обновлением

**Фаза 2 - Расширение источников**
- Новые источники: SteamDB, Epic Games Store, RAWG, Itch.io, Reddit, YouTube, Google Trends
- Расширение dbt слоя: когортный анализ, lifecycle метрики жанров
- Crunchbase + SEC EDGAR для обогащения данными о студиях

**Фаза 3 - Миграция в облако**
- Триггер: публичный ассистент требует облачную инфраструктуру
- ClickHouse -> AWS (S3 + Redshift Serverless) или ClickHouse Cloud
- Airflow -> MWAA или оставить локально
- Tableau Public -> Superset или Tableau Server

**Фаза 4 - Intelligence Layer**
- MCP сервер для Redshift / ClickHouse Cloud
- Публичный BI ассистент: строгая цепочка без LangChain, dbt схема как контекст, генерация SQL под капотом
- Ответ в виде графика или таблицы в зависимости от размера результата
- Интеграция с Continue.dev для помощи в разработке проекта внутри IDE

## Ограничения

- SteamSpy предоставляет оценки, а не реальные данные о продажах
- Прокси выручки приблизителен: owners x price
- Корреляция между метриками не означает причинно-следственную связь
- Доступность данных зависит от настроек приватности профилей Steam
